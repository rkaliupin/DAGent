## Auth

### Data-Plane (Azure Services)

- All data-plane calls use `DefaultAzureCredential`. **Zero API keys in code.**
- Use `getBearerTokenProvider(credential, "<scope>")` for Azure service authentication.

### Application Auth (Dual-Mode)

The app uses a feature-flagged dual-mode auth system controlled by `AUTH_MODE` / `NEXT_PUBLIC_AUTH_MODE`. Demo mode is required for pipeline integration testing (`integration-test` and `live-ui` agents authenticate with demo credentials). Entra ID mode is the production target:

| Mode | Backend | Frontend | API Header |
|------|---------|----------|------------|
| `demo` | POST /auth/login validates shared credentials, returns token | DemoAuthContext + sessionStorage | `X-Demo-Token: <token>` |
| `entra` | Login endpoint returns 404 (disabled) | MSAL v5 redirect to Entra ID | `Authorization: Bearer <jwt>` |

### Demo Credentials

- Username: `demo`
- Password: `demopass`
- Token: UUID generated by Terraform (`random_uuid`), stored in Key Vault

### Switching Modes

To switch from demo to Entra ID:
1. Set `AUTH_MODE=entra` on the backend (Terraform `var.auth_mode`)
2. Set `NEXT_PUBLIC_AUTH_MODE=entra` on the frontend
3. Configure `NEXT_PUBLIC_ENTRA_CLIENT_ID` and `NEXT_PUBLIC_ENTRA_TENANT_ID`
4. APIM policies automatically switch from `check-header` to `validate-jwt`
